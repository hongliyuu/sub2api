// How it works

三步开干

从部署到发第一个请求，五分钟。

1

部署服务

Docker 一键启动，或部署到 Cloud Run / Vercel。支持 PostgreSQL + Redis，也支持 SQLite 轻量模式。

$ docker compose up -d
2

配置账号

在管理后台添加你的 AI 订阅账号，创建分组和 API key。支持 OAuth 授权和手动 token 导入。

admin panel → accounts → add
3

开始使用

把 Base URL 和 API key 填进你的工具（Cursor / Claude Code / 自己的代码），完事。

ANTHROPIC_BASE_URL=https://your.nb.api

// FAQ

常见问题

nbAPI 是一个代理层，你可以把多个订阅账号汇成一个池子，通过统一的 API key 调用。好处是：故障自动转移、负载均衡、多人共享、统一计费。如果你只用一个账号给自己用，可能不需要；但如果你有团队、有多个账号、或想分摊成本 —— nbAPI 就是为你做的。
目前支持 Claude（Anthropic）、GPT（OpenAI）、Gemini（Google）、Antigravity。兼容 OpenAI 协议，所有支持自定义 Base URL 的工具都能直接用。
拼车共享的是订阅额度，不是对话内容。每个用户有独立的 API key，请求互相隔离。nbAPI 不存储任何对话内容，只记录用量信息用于计费。你的数据直接到上游 AI 平台，中间不留存。
最低配置：1核 1G 内存的 VPS 就够了。支持 Docker 部署（推荐）、直接二进制运行、或部署到 GCP Cloud Run 等 Serverless 平台。数据库支持 PostgreSQL（生产推荐）或 SQLite（开发/轻量使用）。
