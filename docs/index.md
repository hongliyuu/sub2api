## Project Overview

Sub2API is an AI API gateway platform for distributing subscription quota through managed API keys. It combines a Go backend for gateway, SaaS, billing, payment, operations and persistence with a Vue frontend for user and admin workflows.

This documentation was generated by BMAD `document-project` using Quick Scan. It is intended as an AI retrieval entry point for brownfield planning and implementation.

## Project Structure

### Backend (`backend`)

- **Type:** Backend/API gateway
- **Location:** `backend`
- **Stack:** Go 1.26.2, Gin 1.9.1, Ent 0.14.5, Wire 0.7.0, PostgreSQL, Redis
- **Entry Point:** `backend/cmd/server/main.go`
- **Pattern:** handler -> service -> repository layered backend

### Frontend (`frontend`)

- **Type:** Web frontend
- **Location:** `frontend`
- **Stack:** Vue 3.5.26, Vite 5.4.21, TypeScript 5.6.3, Pinia, Vue Router, TailwindCSS
- **Entry Point:** `frontend/src/main.ts`
- **Pattern:** Vue SPA with route views, domain components, Pinia stores and Axios API modules

## Cross-Part Integration

- Frontend API modules call backend business APIs through `frontend/src/api/client.ts` and `/api/v1`.
- Vite builds frontend assets into `backend/internal/web/dist`.
- Release builds use Go `-tags embed` so the backend binary can serve the SPA.
- External AI clients call backend gateway endpoints directly through `/v1`, `/v1beta`, `/responses`, `/backend-api/codex`, and `/antigravity`.
