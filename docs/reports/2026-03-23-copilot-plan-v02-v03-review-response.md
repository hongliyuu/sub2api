## 问题核实与处置

### M1（round 2）：M2 处置结论与方案细节存在冲突

**核实结论：Codex 正确。**

v0.2 中存在三处表述不一致：

| 位置 | 表述 |
|------|------|
| v0.2 摘要第 10 行 | "无缓存时保留原有 5xx" |
| v0.2 策略表第 208 行 | "缓存过期 + 上游失败 + 无任何缓存 → 200，静态默认列表" |
| 回应报告汇总表第 123 行（M2 行） | "无缓存时保留 5xx" |

摘要和回应报告的表述（"保留 5xx"）与策略表的实际行为（200 + 静态默认列表）相反。

**决策过程**：

两种策略的取舍：

- **高可用优先**（策略表行为）：无缓存 + 上游失败 → 200 + 静态默认列表。客户端（Claude Code、Codex）不因 `/models` 临时故障而中断；静态默认列表足够让客户端正常工作；warn 日志保障监控感知。
- **语义优先**：无缓存 + 上游失败 → 502。监控信号清晰但客户端会初始化失败。

对 `/models` 这个纯查询端点，客户端强依赖其成功响应（用于模型发现）。高可用优先更合适，且 warn 日志已保障可观测性。**确认采用高可用优先策略**，策略表（第 208 行）的行为是正确的。

**v0.3 修订**：
- 删除"无缓存时保留原有 5xx"的错误表述
- v0.3 摘要改为："**高可用优先**；仅`无账号且无缓存`返回 503；其余故障优先返回 stale 缓存，若无缓存则返回静态默认列表（200）"
- 策略表保持不变（原本即是正确的）
- 回应报告中的汇总表已同步修正（不再出现"保留 5xx"的矛盾描述）
- 唯一返回 503 的场景明确限定为："无可用账号 + 无任何缓存"（真正不可服务）
